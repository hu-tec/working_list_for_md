# vibecoding 화면 분석 쉬운 설명

작성일: 2026-05-21  
분석 대상: `figma_pdf/vibecoding`  
분석 기준: 폴더 내 파일 목록, `기본화면_파일업로드.jpg`, `Frame 758532606.jpg` 확인

## 1. 한 줄 요약

`vibecoding`은 전체 시험 사이트를 모두 설계한 자료가 아니라, **관리자가 많은 회원/시험 데이터를 한 화면에서 보고, 조건별로 걸러보고, 엑셀이나 파일로 일괄 처리하는 관리자용 대량 관리 화면**에 가깝다.

쉽게 말하면, 쇼핑몰 관리자 페이지에서 주문 목록을 한꺼번에 보고 처리하듯이, 이 화면은 시험 플랫폼의 회원, 시험, 결제, 채점, 결과, 피드백 정보를 한 번에 모아 관리하려는 화면이다.

## 3. 이 화면은 누가 쓰는가?

이 화면의 주 사용자는 일반 수험생이 아니라 **관리자**다.

관리자는 다음과 같은 일을 해야 한다.

| 관리자 업무 | 화면에서 보이는 근거 |
|---|---|
| 회원을 검색한다 | 상단 검색창, 회원관리 제목 |
| 회원을 조건별로 걸러본다 | 정회원, 일반회원, 전문가 여부, 성별, 가입경로, 알림 수신 등 체크박스 |
| 시험 정보를 확인한다 | 시험명, 시험유형, 시험상태, 시험결과 관련 컬럼 |
| 결제/접수 상태를 본다 | 접수구분, 시험접수기간, 결제/정산 관련 탭 |
| 대량 데이터를 엑셀로 만든다 | `엑셀파일생성` 버튼 |
| 파일을 올리거나 내려받는다 | `파일올리기`, `파일다운로드` 버튼 |
| 선택한 사람에게 연락한다 | `SMS전송`, 알림 설정 컬럼 |

즉, 이 화면은 “사용자가 시험을 보는 화면”이 아니라 “운영자가 수많은 데이터를 관리하는 화면”이다.

## 4. 화면을 쉽게 풀어 설명

`기본화면_파일업로드.jpg`를 보면 화면은 크게 여섯 영역으로 나뉜다.

| 영역 | 쉬운 설명 |
|---|---|
| 왼쪽 아이콘 메뉴 | 홈, 회원, 학습/시험, 설정 같은 주요 관리 메뉴로 이동하는 곳 |
| 상단 검색/계정 영역 | 검색창, 관리자 이름, 로그아웃이 있는 곳 |
| 페이지 제목과 탭 | 현재 `회원관리` 화면이며, 전체/회원관리/활동관리/결제관리/1:1문의 관리/통계 탭으로 구분됨 |
| 검색 조건 영역 | 원하는 사람이나 시험만 찾기 위해 체크박스, 카테고리, 날짜, 검색어를 선택하는 곳 |
| 전체 정보 테이블 | 회원 정보와 시험 정보를 한 줄씩 보여주는 핵심 목록 |
| 하단/우측 버튼 | 선택삭제, 메일 보내기, SMS 전송, 파일 다운로드 같은 일괄 작업 버튼 |

## 5. 탭 구조

화면 상단에는 다음 탭이 있다.

| 탭 | 의미 |
|---|---|
| 전체 | 회원, 활동, 결제, 문의, 통계를 한 화면에서 통합 조회 |
| 회원관리 | 회원 기본 정보 관리 |
| 활동관리 | 회원의 시험 접수, 응시, 결과, 활동 이력 관리 |
| 결제관리 | 결제 상태, 결제 내역, 정산 정보 관리 |
| 1:1 문의 관리 | 회원 문의와 답변 상태 관리 |
| 통계 | 회원/시험/결제/결과 데이터를 통계로 확인 |

이 탭 구조는 “한 사람을 기준으로 관련 정보를 모두 보는 관리자 화면”에 적합하다. 예를 들어 어떤 회원을 찾은 뒤, 그 회원이 어떤 시험을 봤는지, 결제는 했는지, 문의는 남겼는지, 결과는 어떤지를 한 화면에서 이어서 볼 수 있다.

## 6. 검색 조건의 의미

검색 조건 영역에는 체크박스가 매우 많다. 일반 사용자에게 쉽게 설명하면, 이 부분은 “원하는 사람만 골라보기 위한 필터”다.

예시는 다음과 같다.

| 조건 종류 | 화면에서 보이는 예 | 의미 |
|---|---|---|
| 회원 구분 | 정식회원, 일반회원, 전문가 | 어떤 종류의 회원인지 선택 |
| 성별 | 남, 여 | 성별 기준으로 조회 |
| 회원 등급/상품 | 프리미엄, 스탠다드, 베이직, 없음 | 회원이 어떤 상품 또는 등급을 사용하는지 확인 |
| 알림 수신 | 문자, 이메일, 카카오톡, 없음 | 어떤 연락 수단을 허용했는지 확인 |
| 가입 경로 | 네이버, 카카오, 구글 | 어떤 경로로 가입했는지 확인 |
| 에디터 유형 | 문서생성 에디터, 영상 에디터, 기타 에디터 등 | 회원이 사용하는 작업 도구 또는 기능 유형 |
| 시험 상태 | 검증요청, 수정요청, 게시완료, 취소 | 시험 또는 작업의 진행 상태 |
| 날짜 | 시작일, 종료일 | 특정 기간의 데이터만 조회 |
| 카테고리 | 대/중/소 카테고리 | 시험이나 작업의 분류 기준 |

체크박스 PNG가 많이 들어 있는 이유도 이 화면의 필터 조건이 많기 때문이다.

## 8. 파일 업로드 기능의 의미

파일명에 `파일업로드`가 들어가고, 화면 오른쪽에 `파일올리기`, `파일다운로드`, `엑셀파일생성` 버튼이 보인다.

이 기능은 다음 용도로 쓰일 가능성이 높다.

| 기능 | 쉬운 설명 |
|---|---|
| 엑셀 업로드 | 회원 목록이나 시험 접수 목록을 엑셀로 한 번에 등록 |
| 엑셀 다운로드 | 현재 조회된 목록을 엑셀 파일로 내려받기 |
| 엑셀파일생성 | 선택한 데이터로 관리용 엑셀 파일 생성 |
| 파일올리기 | 시험 자료, 회원 자료, 결과 자료 등을 업로드 |
| 파일다운로드 | 등록된 자료나 결과 파일을 다운로드 |

일반적인 관리자 업무에서는 수백 명의 회원이나 시험 접수자를 하나씩 입력하기 어렵다. 그래서 엑셀 파일을 올려 한 번에 처리하거나, 현재 목록을 엑셀로 내려받아 검토하는 기능이 필요하다.

## 9. `구분` 화면들의 의미

`구분.pdf`, `구분-1.pdf`부터 `구분-14.pdf`까지 많은 PDF가 있다. 이름만 보면 모두 같은 “구분” 계열이다.

여기서 `구분`은 다음 두 가지 의미로 해석할 수 있다.

| 해석 | 설명 |
|---|---|
| 분류 기준 | 회원, 시험, 결제, 문의, 통계 등을 나누는 기준 |
| 테이블 구간 | 너무 넓은 테이블을 여러 부분으로 나눠 보여주는 구간 |

`Frame 758532606.jpg`가 아주 긴 가로 테이블을 보여주기 때문에, `구분-*` PDF들은 이 긴 테이블의 컬럼 묶음을 나눠 저장한 산출물일 가능성이 높다.

쉽게 말하면, 하나의 엑셀 파일에 열이 너무 많아서 가로로 길게 펼쳐진 상태라고 보면 된다. `구분-1`, `구분-2` 같은 파일은 그 엑셀의 일부 구간을 잘라낸 화면일 수 있다.

## 10. 2024 자료와의 관계

`vibecoding`은 2024 전체 사이트맵 중 일부 기능만 떼어낸 것처럼 보인다.

| 2024 전체 자료 | vibecoding 자료 |
|---|---|
| 시험 플랫폼 전체 화면 포함 | 파일 업로드/대량 테이블 관리 중심 |
| 수험생, 출제자, 채점자, 관리자 화면 모두 포함 | 관리자 화면 중심 |
| 로그인, 회원가입, 시험응시, 출제, 채점, 결제, 문의 등 전체 범위 | 회원관리, 활동관리, 결제관리, 문의관리, 통계를 한 화면에 통합 |
| 여러 업무 화면이 나뉘어 있음 | 한 화면에 많은 정보를 모아 보는 방식 |

따라서 `vibecoding`은 독립된 새 사이트라기보다, **관리자가 데이터를 더 빠르게 보고 처리하기 위한 고밀도 관리자 화면 실험안**으로 보는 것이 적절하다.

## 11. 이 화면의 장점

| 장점 | 설명 |
|---|---|
| 한 화면에서 많은 정보를 볼 수 있음 | 회원, 시험, 접수, 결과, 피드백 등을 한 테이블에서 확인 |
| 조건 검색이 강함 | 체크박스와 카테고리로 원하는 대상만 빠르게 조회 |
| 대량 작업에 적합함 | 선택삭제, 메일, SMS, 엑셀 생성, 파일 업로드/다운로드 가능 |
| 운영자가 업무 흐름을 빠르게 파악 가능 | 시험 상태, 결과, 미응시, 합격/불합격 등을 목록에서 바로 확인 |
| 엑셀 기반 업무와 잘 맞음 | 실제 운영 현장에서 자주 쓰는 엑셀 업로드/다운로드 흐름과 연결 |

## 12. 이 화면의 위험하거나 어려운 점

이 화면은 관리자에게는 강력하지만, 그대로 만들면 사용하기 어려울 수도 있다.

| 문제 | 쉬운 설명 | 개선 방향 |
|---|---|---|
| 컬럼이 너무 많음 | 한 줄에 정보가 너무 많아 읽기 어렵다 | 컬럼 그룹화, 고정 컬럼, 보기 설정 필요 |
| 체크박스가 너무 많음 | 처음 보는 사람은 어떤 조건을 골라야 할지 헷갈린다 | 자주 쓰는 필터와 상세 필터를 분리 |
| 가로 스크롤이 길다 | 오른쪽 끝 정보를 보려면 계속 이동해야 한다 | 회원정보/시험정보/결제정보/결과정보 탭 분리 |
| 권한 관리가 중요함 | 개인정보와 결제정보가 한 화면에 함께 보인다 | 관리자 권한별로 볼 수 있는 컬럼 제한 |
| 실수 위험이 있음 | 선택삭제, 파일 업로드 같은 기능은 대량 변경을 일으킨다 | 확인 팝업, 작업 이력, 되돌리기 정책 필요 |

## 13. 일반 사용자를 위한 비유

이 화면은 일반적인 웹사이트 화면이라기보다 **관리자용 엑셀 대시보드**에 가깝다.

예를 들어 학원에서 수강생을 관리한다고 생각하면 쉽다.

1. 수강생 목록을 본다.
2. 정회원인지, 결제를 했는지, 시험을 봤는지 확인한다.
3. 특정 조건의 학생만 검색한다.
4. 필요한 학생에게 문자나 메일을 보낸다.
5. 엑셀로 내려받아 보고서를 만든다.
6. 새 명단을 파일로 올려 한 번에 반영한다.

`vibecoding` 화면도 이와 비슷하다. 다만 대상이 수강생이 아니라 시험 플랫폼의 회원, 시험 접수자, 응시자, 출제/채점/결제 데이터라는 점이 다르다.

## 14. 기능 정의

이 시안을 실제 기능으로 정의하면 다음과 같다.

| 기능명 | 설명 |
|---|---|
| 통합 회원 조회 | 회원 기본정보, 가입정보, 연락처, 알림 설정을 조회 |
| 조건 필터 | 회원 유형, 성별, 가입경로, 상품/등급, 시험 상태, 날짜 등으로 검색 |
| 통합 활동 조회 | 회원의 시험 접수, 응시, 결과, 피드백 정보를 함께 조회 |
| 결제 연계 조회 | 결제 상태, 정산 상태, 환불 상태를 확인 |
| 문의 연계 조회 | 회원의 1:1 문의 상태를 확인 |
| 대량 선택 | 여러 행을 체크해서 한 번에 처리 |
| 선택 삭제 | 선택한 목록을 삭제 또는 제외 처리 |
| 메일/SMS 발송 | 선택한 회원에게 안내 발송 |
| 엑셀 생성 | 현재 데이터 또는 선택 데이터를 엑셀로 생성 |
| 파일 업로드 | 외부 파일을 올려 데이터를 등록/갱신 |
| 파일 다운로드 | 조회 결과 또는 첨부 파일 다운로드 |
| 상태 배지 표시 | 합격, 불합격, 접수완료, 미응시 같은 상태를 색상으로 표시 |

## 15. 구현할 때 추천하는 화면 구조

현재 시안은 한 화면에 모든 정보를 보여주려는 성격이 강하다. 실제 서비스에서는 다음처럼 나누면 더 이해하기 쉽다.

| 추천 화면 | 포함 내용 |
|---|---|
| 회원 기본정보 탭 | 이름, 연락처, 가입일, 회원유형, 알림설정 |
| 시험/활동 탭 | 접수 시험, 시험일, 응시상태, 시험결과 |
| 결제 탭 | 결제금액, 결제수단, 환불, 정산 |
| 문의 탭 | 1:1문의, 답변상태, 문의일 |
| 통계 탭 | 응시율, 합격률, 매출, 회원 증가 |
| 파일관리 탭 | 엑셀 업로드, 다운로드, 업로드 이력 |

이렇게 나누면 초보 관리자도 화면을 이해하기 쉽고, 개인정보와 결제정보 접근 권한도 더 안전하게 나눌 수 있다.

## 16. 최종 판단

`vibecoding`은 2024 전체 사이트의 대체안이 아니라, **관리자 업무를 빠르게 처리하기 위한 고밀도 통합 관리 화면**이다.

핵심은 다음 세 가지다.

1. 많은 데이터를 한 번에 본다.
2. 원하는 조건으로 빠르게 걸러본다.
3. 엑셀/파일/SMS/메일로 대량 처리한다.

따라서 이 자료는 사이트맵의 최상위 기준으로 쓰기보다는, 2024 관리자 화면 중 `회원관리`, `활동관리`, `결제관리`, `1:1문의 관리`, `통계`, `파일 업로드` 기능을 구현할 때 참고하는 세부 관리자 화면 시안으로 쓰는 것이 적절하다.

## 18. UX/UI 관점 보완

`vibecoding` 화면은 기능이 많고 정보량이 매우 크다. 숙련된 관리자에게는 빠르지만, 처음 쓰는 관리자에게는 복잡하게 느껴질 가능성이 높다. 따라서 “모든 정보를 한 번에 보여주는 화면”이 아니라 “필요한 정보를 단계적으로 펼치는 화면”으로 다듬는 것이 좋다.

### 18.1 정보 구조 개선

| 현재 문제 | 개선 방향 |
|---|---|
| 한 테이블에 회원, 시험, 결제, 결과, 피드백이 모두 들어감 | 컬럼 그룹을 `회원정보`, `시험정보`, `결제정보`, `결과정보`, `문의/피드백`으로 나눈다. |
| 가로 스크롤이 매우 김 | 회원명, 회원번호, 상태 같은 핵심 열은 왼쪽에 고정한다. |
| 필터 체크박스가 너무 많음 | 기본 필터와 상세 필터를 접고 펼칠 수 있게 한다. |
| 상태값이 많아 의미 파악이 어려움 | 색상 배지와 짧은 설명을 함께 제공한다. |
| 대량 작업 버튼이 한곳에 모여 있음 | 위험도에 따라 `조회`, `발송`, `파일`, `삭제` 작업을 분리한다. |

### 18.2 관리자 사용 흐름

1. 검색어 또는 기본 필터로 대상을 찾는다.
2. 필요한 컬럼 그룹만 켠다.
3. 목록에서 상태 이상 또는 미처리 항목을 확인한다.
4. 개별 상세를 열어 정확한 내용을 확인한다.
5. 필요한 경우 선택 항목에 대해 메일/SMS/파일 다운로드를 수행한다.
6. 대량 변경이 필요한 경우 업로드 전 미리보기와 검증 결과를 확인한다.

### 18.3 파일 업로드 UX

파일 업로드 기능은 실수하면 많은 데이터가 한 번에 바뀔 수 있으므로 안전장치가 필요하다.

| 단계 | 필요한 화면 요소 |
|---|---|
| 업로드 전 | 허용 파일 형식, 샘플 파일 다운로드, 필수 컬럼 안내 |
| 업로드 직후 | 파일명, 행 수, 오류 행 수, 중복 데이터 수 표시 |
| 반영 전 | 변경될 데이터 미리보기, 추가/수정/삭제 건수 구분 |
| 반영 확인 | 최종 확인 팝업, 관리자 비밀번호 또는 2차 확인 |
| 반영 후 | 처리 결과, 실패 항목 다운로드, 작업 이력 저장 |

### 18.4 접근성 및 개인정보

| 항목 | 권장 기준 |
|---|---|
| 개인정보 | 기본 목록에서는 일부 마스킹하고, 상세 권한이 있는 관리자만 전체 보기 |
| 결제정보 | 결제 금액과 상태는 표시하되 결제수단 상세는 권한별 제한 |
| 시험결과 | 합격/불합격과 점수는 역할 권한에 따라 노출 |
| 대량 다운로드 | 다운로드 사유 입력과 로그 기록 |
| SMS/메일 발송 | 발송 대상 수, 발송 내용 미리보기, 테스트 발송 제공 |
