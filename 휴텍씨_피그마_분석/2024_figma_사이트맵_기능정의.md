# 2024 Figma PDF 기준 사이트맵 및 기능 정의

작성일: 2026-05-21  
분석 대상: `figma_pdf/2024`  
분석 기준: PDF/이미지 파일명, `메뉴 구조.png`, `관리자 홈.png`, 기존 비교 문서의 2024 범위

## 1. 전체 서비스 개요

2024 Figma 자료는 AITP 시험 플랫폼의 전체 업무 흐름을 담고 있다. 단순한 시험 접수 사이트가 아니라, 수험생이 시험을 신청하고 응시한 뒤 결과와 자격증을 받는 흐름, 출제자가 문제를 만들고 제출하는 흐름, 채점자가 답안을 평가하는 흐름, 관리자가 회원·시험·문제·결제·운영을 통제하는 흐름이 함께 설계되어 있다.

일반 사용자가 이해하기 쉽게 말하면, 이 서비스는 다음 네 종류의 사용자를 위한 통합 시험 운영 시스템이다.

| 사용자 | 하는 일 | 대표 화면 |
|---|---|---|
| 수험생 | 시험 접수, 결제, 시험 응시, 결과 확인, 자격증 신청, 문의 | `목록_시험접수.pdf`, `목록_시험응시.pdf`, `목록_시험결과.pdf`, `수험생_자격증 신청하기.png` |
| 출제자 | 시험 문제 작성, 임시저장, 출제내역 확인, 반려 문항 수정 | `출제관리.pdf`, `출제관리_번역상세.pdf`, `출제관리_프롬프트상세.pdf`, `출제관리_단답형상세.pdf` |
| 채점자 | 배정된 답안 채점, 채점내역 확인, 피드백 작성 | `채점관리.pdf`, `채점관리_번역상세.pdf`, `채점관리_프롬프트상세.pdf`, `채점자 홈_시험 채점하기.png` |
| 관리자 | 회원, 시험 일정, 문제, 채점, 결제, 정산, 게시판, 문의, 통계, 코드 관리 | `관리자 홈.png`, `회원관리.pdf`, `시험관리.pdf`, `결제관리.pdf`, `1_1 문의관리.pdf` |

## 2. 상위 사이트맵

`메뉴 구조.png` 기준으로 상단 메뉴는 다음과 같이 구성된다.

| 상단 메뉴 | 주요 사용자 | 설명 |
|---|---|---|
| 관리자 홈 | 관리자 | 전체 운영 현황과 주요 관리 기능 진입점 |
| 회원관리 | 관리자 | 수험생, 출제자, 채점자, 관리자 계정 관리 |
| 시험관리 | 관리자, 수험생 | 시험 일정, 시험 응시, 시험 결과, 자격증 관리 |
| 문제관리 | 관리자, 출제자 | 출제, 출제내역, 문제 검수, 시험 결과 연계 |
| 채점관리 | 관리자, 채점자 | 채점 배정, 채점 진행, 채점 결과 확인 |
| 결제관리 | 관리자, 수험생 | 결제정보, 카드정보, 정산정보, 자격증 정산 |
| 코드추가 | 관리자 | 시험 분류, 언어, 등급, 대/중/소분류 코드 관리 |
| 1:1 문의 관리 | 관리자, 전체 사용자 | 문의 접수, 답변, 문의 상세 확인 |
| 내 정보 | 전체 사용자 | 계정 정보, 비밀번호, 프로필, 로그아웃 |

역할별 홈 메뉴는 다음과 같다.

| 역할 | 홈에서 바로 접근하는 기능 |
|---|---|
| 수험생 홈 | 시험응시, 시험결과, 자격증신청, 1:1문의, 내 정보 |
| 출제자 홈 | 시험 출제하기, 출제 내역, 1:1문의, 내 정보 |
| 채점자 홈 | 시험 채점하기, 채점내역, 1:1문의, 내 정보 |
| 관리자 홈 | 회원관리, 시험관리, 문제관리, 채점관리, 결제관리, 운영관리, 통계관리, 문의관리 |

## 3. 인증 및 계정 기능

사용자가 서비스를 이용하기 전 또는 계정 정보를 관리할 때 필요한 기능이다.

| 기능 | 화면 파일 | 기능 설명 |
|---|---|---|
| 로그인 | `로그인.pdf` | 아이디/비밀번호로 로그인하고 권한에 맞는 홈으로 이동 |
| 로그아웃 | `로그아웃.pdf`, `로그아웃-1.pdf` | 현재 세션 종료 |
| 회원가입 | `회원가입.pdf`, `회원가입_간편가입.pdf` | 일반 가입 또는 간편 가입 |
| 본인/휴대폰 인증 | `본인인증 2.pdf`, `휴대폰 인증.pdf` | 가입, 아이디 찾기, 비밀번호 재설정 전 본인 확인 |
| 아이디 찾기 | `아이디찾기.pdf`, `아이디확인.pdf` | 인증 후 계정 ID 확인 |
| 비밀번호 찾기/재설정 | `비밀번호찾기.pdf`, `비밀번호재설정.pdf`, `비밀번호설정완료.pdf` | 새 비밀번호 설정 |
| 회원탈퇴 | `회원탈퇴.pdf`, `회원탈퇴-1.pdf`, `회원탈퇴_환불.pdf`, `회원탈퇴_환불-1.pdf` | 탈퇴 전 환불/보존 정보 확인 후 탈퇴 처리 |
| 이용약관 | `이용약관.pdf` | 가입 및 서비스 이용 약관 확인 |

핵심 정책은 계정 권한에 따라 화면 접근이 달라져야 한다는 점이다. 같은 로그인 화면을 사용하더라도 수험생은 시험 접수와 응시 중심, 출제자는 출제 작업 중심, 채점자는 채점 작업 중심, 관리자는 운영 관리 중심으로 진입해야 한다.

## 4. 수험생 영역

수험생 영역은 “시험을 찾고 접수한다 → 결제한다 → 시험을 본다 → 결과를 확인한다 → 자격증을 신청한다”의 흐름으로 이해하면 된다.

### 4.1 시험 접수

| 화면 | 기능 |
|---|---|
| `목록_시험접수.pdf` | 접수 가능한 시험 목록 확인 |
| `목록_시험접수_팝업.pdf` | 시험 접수 전 상세 확인 또는 신청 팝업 |
| `목록_시험접수_변경.pdf` | 접수 정보 변경 |
| `목록_시험접수_변경확인.pdf` | 변경 내용 최종 확인 |
| `목록_시험접수_취소.pdf` | 접수 취소 및 환불 가능 여부 확인 |

주요 기능은 시험명, 차수, 등급, 언어, 시험일, 접수기간, 응시료, 접수상태를 확인하고 접수/변경/취소를 처리하는 것이다.

### 4.2 시험 응시

| 화면 | 기능 |
|---|---|
| `목록_시험응시.pdf`, `목록_시험응시-1.pdf` | 응시 가능한 시험 목록 확인 |
| `응시생_시험시작.png` | 시험 시작 전 진입 화면 |
| `응시생_시험중.png` | 실제 시험 진행 화면 |
| `응시생_시험중_문장나누기).png` | 문장 분할형 시험 진행 |
| `응시생_시간확인.png` | 남은 시간 확인 |
| `응시생_전체화면.png` | 시험 중 전체화면 상태 |
| `응시생_제출.png`, `응시생_제출확인.png` | 답안 제출 및 제출 확인 |
| `응시생_종료.png` | 시험 종료 화면 |
| `응시생_다음차시.png` | 다음 차시 이동 |

시험 응시 화면은 부정행위 방지, 시간 제한, 답안 저장, 제출 확인이 중요하다. 특히 시험 도중에는 현재 문항, 남은 시간, 다음/이전 이동, 임시저장 또는 자동저장, 최종 제출 상태가 명확해야 한다.

### 4.3 시험 유형별 응시

| 유형 | 화면 | 설명 |
|---|---|---|
| 번역형 | `시험_번역(번역기지정).pdf`, `시험_출제_번역기선택(텍스트)_전체보기.pdf` | 번역기 선택, 원문/번역문 작성, 전체보기 |
| 음성/영상 번역 | `시험_음성영상(번역기2개).pdf`, `시험_음성영상(번역기3개).pdf`, `영상.pdf` | 음성·영상 자료 기반 응시 |
| 웹툰 번역 | `시험_웹툰번역(번역기선택).png` | 이미지/웹툰 콘텐츠 번역 |
| 프롬프트형 | `응시생_프롬프트AI선택.png`, `응시생_프롬프트설정.png`, `응시생_프롬프트시작.png`, `응시생_프롬프트에디터.png` | AI 선택, 프롬프트 조건 설정, 에디터 작성 |
| 단답형 | `응시생_단답형시작.png`, `응시생_단답형시험.png`, `상세_단답형.pdf` | 짧은 답안 입력형 시험 |

### 4.4 시험 결과 및 자격증

| 화면 | 기능 |
|---|---|
| `목록_시험결과.pdf` | 응시한 시험의 결과 목록 확인 |
| `목록_자격증신청.pdf` | 합격 후 자격증 신청 목록 확인 |
| `수험생_자격증 신청하기.png` | 자격증 발급 신청 |

결과 화면에서는 합격/불합격, 점수, 채점중 여부, 성적표 보기, 자격증 신청 가능 여부가 표시되어야 한다. 자격증 신청은 합격자에게만 활성화되어야 하며 결제 또는 배송 정보와 연결될 수 있다.

## 5. 출제자 영역

출제자 영역은 문제를 새로 만들고, 임시저장하고, 제출하고, 승인/반려 상태를 확인하는 업무 공간이다.

| 화면 | 기능 |
|---|---|
| `출제관리.pdf` | 출제 업무 목록과 검색 |
| `출제관리_번역상세.pdf` | 번역형 문제 상세 작성/확인 |
| `출제관리_프롬프트상세.pdf` | 프롬프트형 문제 상세 작성/확인 |
| `출제관리_단답형상세.pdf` | 단답형 문제 상세 작성/확인 |
| `목록_출제내역.pdf` | 출제자가 제출한 문제 목록 |
| `시험 출제내역.png` | 출제 이력 화면 |
| `상세_출제_설정.pdf`, `상세_출제_설정-1.pdf` | 출제 기본 설정 |
| `상세_프롬프트출제_설정.pdf`, `상세_프롬프트출제_설정-1.pdf` | 프롬프트 출제 조건 설정 |
| `상세_프롬프트출제_확인.pdf` | 프롬프트 출제 최종 확인 |
| `상세_단답형출제_출제.pdf` | 단답형 문항 작성 |
| `상세_단답형출제_확인.pdf` | 단답형 문항 확인 |
| `임시저장.pdf`, `저장.pdf`, `저장팝업.pdf` | 저장/임시저장 확인 |

출제 기능의 핵심 필드는 시험명, 차수, 등급, 언어쌍, 대/중/소분류, 문제 유형, 난이도, 배점, 지문, 문제, 정답, 해설, 평가기준, 첨부파일이다.

출제 프로세스는 다음과 같다.

1. 출제자가 로그인한다.
2. 담당 시험 또는 새 문제 생성을 선택한다.
3. 시험/언어/분류/난이도/문제 유형을 설정한다.
4. 지문, 문제, 정답, 해설, 평가기준을 입력한다.
5. 미리보기로 수험생 화면 기준 표시를 확인한다.
6. 임시저장 또는 최종제출을 한다.
7. 관리자가 검수한다.
8. 반려되면 수정 후 재제출하고, 승인되면 문제은행 또는 시험에 반영된다.

## 6. 채점자 영역

채점자 영역은 배정된 답안을 확인하고, 기준표에 따라 점수와 피드백을 입력하는 업무 공간이다.

| 화면 | 기능 |
|---|---|
| `채점자 홈.png` | 채점자 작업 현황 |
| `채점자 홈_시험 채점하기.png` | 채점 대상 시험 선택 |
| `채점자 홈_시험 채점하기_개별채점하기.png` | 개별 답안 채점 |
| `채점자 홈_채점내역.png` | 완료/진행 중 채점 내역 |
| `채점관리.pdf` | 관리자 또는 채점자용 채점 목록 |
| `채점관리_번역상세.pdf` | 번역형 답안 채점 |
| `채점관리_프롬프트상세.pdf` | 프롬프트형 답안 채점 |
| `채점관리_단답형상세.pdf` | 단답형 답안 채점 |
| `상세_번역채점.pdf`, `상세_번역채점_피드백.pdf` | 번역 채점과 피드백 |
| `상세_프롬프트채점.pdf`, `상세_프롬프트채점_상세.pdf` | 프롬프트 채점 상세 |
| `목록_채점내역.pdf` | 채점 완료/진행 목록 |
| `평가서 팝업.png` | 평가서 또는 채점 의견 팝업 |

채점 화면은 문제, 수험생 답안, 평가 기준, 점수 입력, 코멘트, 임시저장, 채점완료 버튼이 한 흐름으로 보여야 한다. 특히 주관식/번역형/프롬프트형은 자동 채점이 어렵기 때문에 채점 기준표와 이력 추적이 중요하다.

## 7. 관리자 영역

관리자 영역은 전체 플랫폼을 운영하는 본부 화면이다. `관리자 홈.png` 기준으로 검색 조건, 체크박스 필터, 대량 테이블, 상태 버튼, 팝업, 삭제 안내, 시험 추가/삭제 기능이 확인된다.

### 7.1 회원관리

| 화면 | 기능 |
|---|---|
| `회원관리.pdf` | 전체 회원 목록 조회 |
| `회원관리_상세_전체.pdf` | 회원 상세 정보 확인/수정 |
| `그룹관리.pdf` | 회원 그룹 또는 권한 그룹 관리 |

## 11. 구현 관점 우선순위

1. 권한별 라우팅과 공통 레이아웃을 먼저 만든다.
2. 수험생의 시험 접수/결제/응시/결과/자격증 흐름을 연결한다.
3. 출제자 문제 작성과 관리자 문제 검수 흐름을 연결한다.
4. 채점자 채점과 관리자 결과 확정 흐름을 연결한다.
5. 결제/정산/환불을 시험 접수 및 자격증 신청과 연결한다.
6. 고객지원, 게시판, FAQ, 블로그, 사이트관리, 통계를 운영관리로 묶는다.
7. STT/TTS/Youtube/영상/번역 기능은 시험 유형 확장 모듈로 분리한다.

## 12. 주의 사항

현재 문서는 PDF 내부 텍스트 전문 OCR이 아니라, 파일명과 확인 가능한 대표 이미지 기준으로 정리했다. 따라서 세부 컬럼명, 버튼명, 입력값은 실제 PDF 화면을 개별 확인하면서 보완해야 한다. 다만 전체 IA와 기능 범위, 역할별 흐름은 2024 Figma 자료만으로도 명확하게 파악된다.

## 13. UX/UI 관점 보완

2024 화면은 기능 범위가 넓기 때문에 단순히 메뉴를 모두 노출하면 사용자가 길을 잃기 쉽다. 역할별로 “지금 해야 할 일”을 먼저 보여주고, 상세 기능은 그 다음 단계에서 펼쳐야 한다.

| 역할 | 첫 화면에서 가장 먼저 보여줘야 할 것 | UX/UI 설계 방향 |
|---|---|---|
| 수험생 | 접수한 시험, 오늘 응시 가능 여부, 결과 발표 여부 | 상태 카드와 다음 행동 버튼을 크게 배치한다. 예: `결제하기`, `시험시작`, `결과보기`, `자격증신청` |
| 출제자 | 임시저장 문항, 반려 문항, 제출 마감 | `이어쓰기`, `수정 후 재제출`, `새 문제 생성`을 빠르게 누를 수 있게 한다. |
| 채점자 | 채점 대기 건수, 진행 중 채점, 마감 임박 | 채점 대상 목록과 `채점 시작` 버튼을 우선 배치하고, 기준표는 채점 화면에서 고정 영역으로 제공한다. |
| 관리자 | 미처리 업무, 오류/대기 상태, 오늘의 운영 지표 | 대시보드에서 문의 미답변, 결제 미승인, 환불 대기, 검수 대기를 바로 확인하게 한다. |

### 13.1 목록 화면 UX

관리자와 사용자 화면 모두 목록 테이블이 많다. 테이블은 다음 기준을 적용하는 것이 좋다.

| 항목 | 권장 방식 |
|---|---|
| 핵심 열 | 회원명, 시험명, 상태처럼 판단에 필요한 값은 왼쪽에 고정한다. |
| 상태값 | 텍스트만 쓰지 말고 색상 배지와 문구를 함께 표시한다. |
| 주요 버튼 | 한 행에 버튼을 너무 많이 넣지 않고 가장 중요한 1개 행동만 먼저 노출한다. |
| 가로 스크롤 | 핵심 열은 고정하고 세부 정보는 탭 또는 상세 화면으로 분리한다. |
| 검색 조건 | 기본 검색과 상세 검색을 나누어 초보 관리자가 부담을 느끼지 않게 한다. |
| 빈 목록 | `정보 없음`만 보여주지 말고 다음 행동을 안내한다. 예: `등록된 시험이 없습니다. 새 시험을 등록하세요.` |

### 13.2 입력 화면 UX

시험 등록, 출제, 채점, 문의 작성처럼 입력 항목이 많은 화면은 한 페이지에 모든 필드를 노출하면 실수가 늘어난다.

| 화면 | 권장 UX |
|---|---|
| 시험 등록 | 기본정보, 일정, 접수/환불, 시험유형, 공개설정 단계로 나눈다. |
| 문제 출제 | 시험 선택, 문제 작성, 정답/해설, 평가기준, 미리보기, 제출 단계로 나눈다. |
| 채점 | 문제/답안/기준표/점수입력/코멘트를 한 화면에 두되, 기준표와 저장 버튼은 항상 보이게 한다. |
| 결제/환불 | 금액, 환불 가능 여부, 처리 후 결과를 명확히 보여주고 최종 확인 팝업을 둔다. |
| 1:1 문의 | 문의 유형, 관련 시험, 제목, 내용, 첨부파일만 우선 제공한다. |

### 13.3 접근성과 신뢰성

| 항목 | 보완 기준 |
|---|---|
| 색상 | 합격/불합격, 완료/대기, 위험/취소 상태는 색상과 텍스트를 함께 사용한다. |
| 버튼 | 삭제, 환불, 최종제출, 성적공개 같은 되돌리기 어려운 작업은 확인 팝업과 이력 기록을 둔다. |
| 자동저장 | 응시, 출제, 채점 화면에는 자동저장 또는 저장 상태 표시가 필요하다. |
| 시간 표시 | 시험 시간, 접수 마감, 결과 발표일은 날짜와 시간을 함께 명확히 표시한다. |
| 개인정보 | 관리자 테이블에서는 연락처, 이메일, 주소 같은 개인정보를 권한에 따라 마스킹할 수 있어야 한다. |
