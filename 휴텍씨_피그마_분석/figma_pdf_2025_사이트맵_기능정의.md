# figma_pdf/2025 사이트맵 및 기능 정의

작성일: 2026-05-21  
분석 범위: `figma_pdf/2025` 폴더의 화면 파일명, 주요 PNG/JPG 화면 확인, 기존 `figma_pdf_2024_2025_vibecoding_비교분석.md` 참고  
주의: PDF 내부 텍스트 추출 도구가 설치되어 있지 않아, PDF는 파일명과 확인 가능한 메타 정보 중심으로 해석했다. PNG/JPG 화면은 직접 확인했다.

## 1. 한눈에 보는 2025 구성

`figma_pdf/2025`는 전체 시험 플랫폼을 처음부터 끝까지 모두 담은 완전한 IA라기보다, 기존 시험 플랫폼 위에 적용할 **UX/UI 참고 화면과 일부 관리자/전문가 화면**을 묶은 자료로 보는 것이 적절하다.

1. 사용자 경험 참고 영역
   - 창작작업실
   - 메타-T 번역
   - 장바구니
   - 결제
   - 마이페이지 참고 화면

2. 전문가 업무 영역
   - 프롬프트 출제
   - 단답형 출제
   - 출제 내역, 채점 내역, 정산 내역, 전문가 프로필

3. 관리자 운영 영역
   - 대시보드
   - 회원관리
   - 관리자 관리
   - 시험관리
   - 자격증관리
   - 카테고리 관리
   - 양식관리
   - 템플릿 양식 관리자
   - 파일 업로드형 대량 관리 화면

4. 연결 설명 자료
   - `Group 758531575.jpg`
   - `Frame 758532606.jpg`
   - `Rectangle 34625361.pdf`, `Rectangle 34625362.pdf`
   - 다수의 `CheckBox*.png`

특히 `창작작업실01.png`와 `Group 758531575.jpg`는 시험 플랫폼의 신규 핵심 메뉴를 확정하는 자료라기보다, **향후 화면 구성과 상호작용을 참고하기 위한 UX/UI 레퍼런스**로 분류하는 것이 안전하다. `Group 758531575.jpg`는 별도 서비스 화면이 아니라, 관리자/양식 설정 화면의 항목들이 작업 화면과 하단 번역/편집 패널로 어떻게 연결될 수 있는지 설명하는 참고 이미지다.

## 2. 사용자 관점 서비스 설명

일반 사용자가 보는 2025 자료는 "시험 플랫폼"에 바로 붙일 확정 기능이라기보다, 시험 플랫폼의 화면 경험을 확장할 때 참고할 수 있는 AI 작업 도구형 UI 예시를 포함한다.

`창작작업실01.png`는 사용자가 문서, 유튜브, 리서치, 프로그래밍, 이미지/디자인, 음악, 음성 같은 작업 유형을 고르고 파일이나 텍스트를 입력하는 방식을 보여준다. 다만 현재 해석에서는 이 화면을 독립 구현 대상이라기보다, 시험 플랫폼의 향후 UX/UI를 설계할 때 참고할 수 있는 패턴으로 본다.

관리자는 사용자, 시험, 자격증, 카테고리, 양식, 템플릿을 관리한다. 2025에서 새로 강조되는 부분은 기존 시험 관리뿐 아니라 AI 작업에 쓰이는 양식과 템플릿을 운영자가 관리한다는 점이다.

전문가는 프롬프트 출제, 단답형 출제, 채점, 정산, 전문가 프로필 관리를 수행한다. 일반 사용자 화면과 달리 전문가 영역은 문제 작성, 평가요소 설정, 답변 예시 입력처럼 시험/검수 업무 중심이다.

## 4. 사용자 영역 상세 기능

### 4.1 창작작업실 참고 화면

대표 파일: `창작작업실01.png`

창작작업실은 현재 기준에서 사용자가 실제로 진입해야 하는 확정 메뉴라기보다, 2025 자료가 제안하는 작업형 UX/UI 참고 화면이다. 상단에는 `창작작업실`, `요금제`, `고객센터`, `회사소개` 메뉴가 있고, 우측에는 알림, 장바구니, 계정 아이콘이 있다.

작업 유형 탭:

| 탭 | 의미 | 사용 예 |
|---|---|---|
| Youtube | 유튜브 영상 기반 요약, 번역, 분석, 콘텐츠 생성 | 영상 요약, 자막 번역, 영상 내용 분석 |
| 문서 | 문서 파일 기반 작업 | PDF 번역, 문서 요약, 문서 변환 |
| 리서치&분석 | 자료 조사와 분석 | 시장 조사, 법률/논문 분석 |
| 프로그래밍 | 코드 생성/검토/설명 | 코드 작성, 오류 분석, 리팩터링 |
| 이미지,디자인 | 이미지 또는 디자인 관련 작업 | 이미지 설명, 디자인 검토, 시안 생성 |
| 음악 | 음악 관련 작업 | 가사, 음악 분석, 추천 |
| 음성 | 음성 파일 관련 작업 | STT, 요약, 음성 분석 |
| 기타 | 정형화되지 않은 AI 요청 | 일반 질의, 혼합 작업 |

작업 방식 버튼:

| 버튼 | 사용자 의미 | 기능 정의 |
|---|---|---|
| 보유한 양식에 텍스트 입력 | 기존 양식을 선택해 내용을 넣는 방식 | 관리자가 등록한 양식/템플릿을 기반으로 입력 폼을 표시 |
| 나만의 양식 만들기 | 사용자가 직접 양식을 구성 | 필드, 안내문, 옵션을 사용자 정의 |
| 양식 생성 후 내용 작성 | AI 또는 시스템이 양식을 먼저 만든 뒤 내용 입력 | 양식 생성 단계와 작성 단계를 분리 |
| 양식 만들기 | 빈 양식 작성 | 신규 양식 생성 화면으로 이동 |
| 내용 작성 후 검수 받기 | 작성 결과를 전문가/관리자에게 검수 요청 | 후면 작업 요청, 검수비/소요시간 연동 가능 |
| 관련 서비스 표시 영역 | 현재 작업과 관련된 부가 서비스를 보여주는 영역 | 추천 서식, 전문가 서비스, 결제 상품 노출 |

입력/설정 영역:

| 영역 | 기능 |
|---|---|
| 카테고리 선택 | 대분류, 중분류, 소분류를 선택하고 검색한다. 예시는 법률, 국내 법률, 고소장이다. |
| 편집 에디터 | 에디터 사용/미사용을 선택한다. 사용 시 결과 편집 패널과 연결된다. |
| 사용 AI | ChatGPT, Gemini, wrtn., 다른 AI 등 사용할 엔진을 선택한다. |
| 후면 작업 요청 | 검수 요청, 고급전문가, 긴급 여부를 선택한다. |
| 설정 | 기능 사용/미사용을 지정한다. |
| 사용프롬프트 설정 | 전문 분야와 세부분야를 선택한다. |
| 다룰설정 | 관련 내용 체크박스를 선택한다. 실제 서비스에서는 추천 키워드, 검수 기준, 포함 정보 같은 세부 옵션으로 구체화해야 한다. |
| 파일/텍스트 입력 | 파일 업로드 또는 AI 텍스트 입력을 받는다. |
| 추천 서식 | AI 추천 관련 서식을 표시한다. |
| 임시저장/카드/적용 | 임시 저장, 장바구니 담기, 작업 적용을 수행한다. |

### 4.2 메타-T 번역

대표 파일: `메타번역.png`

메타-T 번역은 창작작업실 참고 화면과 비슷한 레이아웃을 쓰지만, 번역 작업에 특화된 화면이다. 이 화면도 시험 플랫폼 본류 기능으로 바로 확정하기보다, 번역/편집 UX 참고 자료로 우선 분류한다.

주요 기능:

| 영역 | 기능 |
|---|---|
| 번역 유형 탭 | 동시통역, 문서번역, YouTube, 기타, 웹툰번역 중 선택한다. |
| 번역 목적 칩 | 아나운서, 관광가이드, 큐레이터, 안내 같은 세부 작업 유형을 선택한다. |
| 카테고리 선택 | 법률, 국내 법률, 고소장 같은 분야를 지정한다. |
| 언어 설정 | 출발어와 도착어를 선택하고, 영어/중국어/일어/아랍어/베트남어 등 언어 옵션을 고른다. |
| 사용 AI | 번역에 사용할 AI 엔진을 선택한다. |
| 양식 에디터 | 결과를 편집기에서 다룰지 선택한다. |
| 후면 작업 요청 | 검수 요청, 전문가 요청, 긴급 여부를 지정한다. |
| 파일/텍스트 입력 | 번역할 파일 또는 텍스트를 입력한다. |
| 관련 서식 추천 | 번역 작업에 맞는 관련 서식을 제안한다. |

### 4.3 번역/편집 작업 화면

대표 근거: `Group 758531575.jpg` 하단 화면

이 화면은 창작작업실 참고 화면 또는 메타-T 번역에서 입력한 자료가 작업 패널로 펼쳐질 수 있음을 보여준다. 원문, AI 번역기, 비교 번역기, 에디터가 나란히 표시된다.

주요 기능:

| 패널 | 기능 |

| AI 번역기 | ChatGPT 등 번역기별 결과 표시, 언어 선택, AI 설정 팝업 |
| 비교 번역기 | 다른 번역 엔진 결과와 비교, 시간/버전 선택 |
| 에디터 | 최종 편집, 문장별 체크, 수정 내용 강조, 서식/링크/인용/설정 도구 사용 |
| 추천 단어 | 추천 단어, 설명, 맞춤법/추천어 후보 표시 |
| 메모 | 작업 중 참고 메모 입력 |
| 저장/결제/적용 | 편집 결과 저장, 결제, 최종 적용 |

### 4.4 장바구니와 결제

대표 파일: `장바구니.png`, `결제.png`

장바구니는 AI 작업 또는 번역 작업을 상품처럼 담아 결제 전 확인하는 화면이다.

장바구니 기능:

| 기능 | 설명 |
|---|---|
| 전체 선택 | 여러 작업을 한 번에 선택한다. |
| 선택 삭제 | 선택한 작업을 장바구니에서 제거한다. |
| 작업 카드 | 작업명, 첨부 파일명, 언어쌍, 번역기, 문서 변환 유형, 에디팅 사용 여부, 글자 수, 가격을 표시한다. |
| 선택 요약 | 선택된 번역 수, 작업명, 단가, 추가 옵션 금액을 보여준다. |
| 총액 계산 | 선택한 작업의 최종 결제 금액을 계산한다. |
| 신용카드/페이 결제 | 일반 결제 수단으로 결제한다. |
| 포인트 결제 | 보유 포인트를 사용한다. |
| 구독 신청 | 정기 구독 상품으로 전환한다. |
| 전문가 견적 요청 | 고급 전문가 작업이 필요한 경우 별도 견적을 요청한다. |

결제 화면은 장바구니의 축약 버전이다. 선택된 작업 1건의 상세와 총 결제 금액, 결제 버튼을 보여준다.

## 5. 전문가 영역 상세 기능

### 5.1 프롬프트 출제하기

대표 파일: `상세_프롬프트출제_설정.png`, `상세_프롬프트출제_확인.png`, `출제관리_프롬프트상세.pdf`

전문가가 프롬프트형 시험 문제를 작성하는 화면이다.

주요 기능:

| 영역 | 기능 |

## 9. 사용자 플로우

### 9.1 창작작업실 참고 플로우

1. 사용자가 작업형 화면에 진입한다고 가정한다.
2. 작업 유형을 선택한다. 예: Youtube, 문서, 리서치&분석.
3. 카테고리를 선택한다. 예: 법률 > 국내 법률 > 고소장.
4. 사용할 AI를 선택한다.
5. 파일 또는 텍스트를 입력한다.
6. 필요하면 검수 요청, 고급전문가, 긴급 작업을 선택한다.
7. 추천 서식을 선택하거나 직접 입력한다.
8. 임시저장, 장바구니 담기, 적용 중 하나를 선택한다.
9. 결과 편집이 필요한 경우 번역/편집 작업 화면으로 이동한다.
10. 작업 완료 후 장바구니 또는 결제로 이동한다.

### 9.2 메타-T 번역 플로우

1. 사용자가 메타-T 번역에 진입한다.
2. 번역 유형을 선택한다. 예: 문서번역.
3. 번역 목적을 선택한다. 예: 큐레이터.
4. 출발어와 도착어를 선택한다.
5. 사용할 AI 번역기를 선택한다.
6. 파일 또는 텍스트를 입력한다.
7. AI 번역 결과와 비교 번역 결과를 확인한다.
8. 에디터에서 최종 번역문을 수정한다.
9. 저장, 결제, 적용을 수행한다.

### 9.3 양식관리와 작업형 참고 화면 연결 플로우

1. 관리자가 카테고리와 양식을 등록한다.
2. 관리자가 양식에 사용할 AI, 입력 필드, 추천 서식, 검수 옵션을 설정한다.
3. 사용자가 작업형 화면에서 해당 카테고리나 양식을 선택한다고 가정한다.
4. 좌측 설정 패널과 우측 입력 영역이 관리자 설정값에 맞춰 표시되는 구조를 참고한다.
5. 사용자가 입력한 작업은 AI 처리, 전문가 검수, 번역/편집, 결제 흐름으로 이어진다.

## 11. 확인이 필요한 미정 항목

| 항목 | 확인 필요 내용 |
|---|---|
| 창작작업실의 성격 | 실제 구현 대상인지, UX/UI 참고 자료인지 확정 필요. 현재 해석은 참고 자료에 가깝다. |
| 양식과 템플릿의 차이 | 양식은 입력 폼, 템플릿은 추천/상품화된 양식으로 보이나 정책 확정 필요 |

| 관리자 대시보드 지표 | 실제 운영 KPI와 표시 우선순위 확인 필요 |

## 12. 요약

2025 구성은 "시험 플랫폼 운영 시스템" 위에 적용할 수 있는 UX/UI 참고 자료와 일부 관리자/전문가 화면을 포함한다. 특히 창작작업실과 메타-T 번역은 현재 기준에서 핵심 구현 기능이라기보다, 사용자 작업 화면과 번역/편집 화면의 참고 레이아웃으로 보는 것이 안전하다.

`Group 758531575.jpg`는 이 연결 구조를 설명하는 자료다. 따라서 별도 화면으로 구현하지 않고, 양식관리와 작업형 참고 화면, 번역/편집 패널 간 연결 관계를 설명하는 근거로 문서화해야 한다.

## 13. UX/UI 관점 보완

2025 화면은 2024의 시험 운영 화면보다 사용자가 직접 작업물을 만드는 성격이 강하다. 따라서 단순 관리자 테이블보다 “입력 → AI 처리 → 결과 확인 → 수정 → 저장/결제” 흐름이 자연스럽게 이어져야 한다.

| 영역 | UX/UI 핵심 |
|---|---|
| 창작작업실 참고 화면 | 작업 유형, 카테고리, AI, 파일/텍스트 입력, 옵션을 순서대로 배치하는 UX/UI 참고 자료로 활용한다. |
| 메타-T 번역 | 원문, AI 번역 결과, 에디터 수정본을 비교하기 쉽게 배치해야 한다. |
| 양식관리 | 관리자가 만든 양식이 사용자 화면에 어떻게 보일지 미리보기로 확인할 수 있어야 한다. |
| 템플릿 양식 관리자 | 템플릿이 어떤 카테고리와 작업 유형에 쓰이는지 명확히 표시해야 한다. |
| 장바구니/결제 | 작업 옵션이 가격에 어떻게 반영되는지 사용자가 이해해야 한다. |

### 13.1 창작작업실 참고 화면 설계 기준

| 화면 요소 | 권장 설계 |
|---|---|
| 작업 유형 탭 | Youtube, 문서, 리서치&분석 등은 상단 탭으로 제공하되 선택된 탭을 명확히 표시한다. |
| 카테고리 선택 | 대/중/소분류가 길어질 수 있으므로 단계형 선택 또는 검색형 셀렉트를 사용한다. |
| AI 선택 | 여러 AI를 동시에 비교할 수 있는지, 하나만 선택하는지 정책을 명확히 한다. |
| 파일 입력 | 업로드 가능 형식, 용량, 처리 상태, 실패 사유를 바로 보여준다. |
| 옵션 | 긴급, 전문가 검수, 고급전문가 같은 비용 발생 옵션은 가격 영향과 함께 표시한다. |
| 실행 버튼 | `임시저장`, `장바구니`, `적용`의 차이를 사용자가 이해할 수 있도록 버튼 위치와 문구를 구분한다. |

### 13.2 번역/편집 UX

| 항목 | 권장 방식 |
|---|---|
| 원문/번역문 | 좌우 또는 상하 분할로 비교 가능하게 배치한다. |
| AI별 결과 | ChatGPT, Gemini 등 AI별 결과를 탭이나 카드로 구분한다. |
| 최종 편집본 | 사용자가 수정한 최종본은 별도 저장 영역으로 분리한다. |
| 변경 이력 | 원문, AI 결과, 사용자 수정본의 차이를 추적할 수 있어야 한다. |
| 저장/결제 | 결과를 저장하는 행동과 결제로 넘어가는 행동을 분리한다. |

### 13.3 관리자 UX

2025 관리자 화면은 기존 시험 운영과 신규 AI 서비스 설정이 섞인다. 관리자가 헷갈리지 않도록 메뉴를 다음처럼 분리하는 것이 좋다.

| 메뉴 그룹 | 포함 기능 |
|---|---|
| 서비스 설정 | 카테고리, 양식관리, 템플릿 양식 관리자 |
| 시험 운영 | 시험관리, 자격증관리, 출제/채점 관리 |
| 사용자 운영 | 회원관리, 전문가 프로필 |
| 판매/결제 | 장바구니, 결제, 요금제, 포인트 |
| 콘텐츠 운영 | 메인 화면, 추천 서식, 안내 문구 |

### 13.4 빈 상태와 오류 상태

| 상태 | 필요한 안내 |
|---|---|
| 파일 없음 | `파일을 업로드하거나 텍스트를 입력하세요.` |
| 파일 형식 오류 | 허용 파일 형식과 최대 용량을 표시한다. |
| AI 처리 실패 | 재시도 버튼과 실패 사유를 제공한다. |
| 결과 없음 | 입력 조건을 다시 확인하도록 안내한다. |
| 결제 전 | 선택한 옵션과 예상 금액을 다시 확인시킨다. |
