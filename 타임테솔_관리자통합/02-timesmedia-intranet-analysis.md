# 타임스미디어 인트라넷 홈페이지 분석

## 1. 분석 개요

| 항목 | 내용 |
| --- | --- |
| 대상 | 타임스미디어 인트라넷 |
| 주소 | `https://gw.timestesol.com/` |
| 확인일 | 2026-05-26, 로그인 이후 HTML 추가 분석 2026-05-28 |
| 분석 범위 | 비로그인 리다이렉트와 로그인 화면, 응답 헤더, `timetesol_source`의 로그인 이후 HTML 55개 |
| 확인 경로 | `/`, `/login.php`, 로그인 이후 스냅샷 HTML |
| 캡처 | [times-login.png](../../.captures/times-login.png) |

초기 주소는 JavaScript로 `/login.php`로 이동시킨다. 최초 분석 시에는 계정이 없어 로그인 이후 기능을 확인하지 못했으나, 이후 `timetesol_source` 폴더에 저장된 로그인 이후 화면 HTML을 기준으로 전체 인트라넷의 목적과 주요 업무 기능을 추가 분석했다.

소스 HTML에는 실제 운영 데이터로 보이는 학생, 직원, 거래처, 회계, 미팅 내용 일부가 포함되어 있다. 본 문서는 개별 개인정보나 민감 업무 내용을 재기재하지 않고, 화면 구조와 기능 관점으로만 요약한다.

## 2. 한줄 진단

타임스미디어 인트라넷은 교육기관 운영에 필요한 학생 모집/상담, 수강·회계, 근태, 전자결재, 공지, 온라인 협업, 출강, 홈페이지 운영을 한곳에 모은 내부 업무 포털이다. 기능 범위는 넓지만, 화면 구조는 고정폭 테이블·팝업·레거시 JavaScript 중심이라 검색성, 반응형 UX, 보안, 개인정보 보호, 유지보수성 개선이 필요하다.

## 3. 사이트 목적

이 사이트의 핵심 목적은 타임스미디어 및 관련 교육 브랜드의 일상 운영을 내부 직원과 강사가 처리하도록 돕는 그룹웨어/학원관리 통합 시스템이다. 단순 사내 게시판이 아니라 다음 업무를 한 시스템 안에서 연결한다.

| 목적 | 설명 |
| --- | --- |
| 교육 상품과 수강생 운영 관리 | 테솔, 번역사, ICS, 국제학교, 기타 과정의 상담·예약·등록·졸업·환불·연기 이력을 조회하고 관리한다. |
| 회계 및 납부 상태 관리 | 입학금, 수업료, 납부 여부, 현금영수증, 환불/연기/전출, 회계 데이터와 센터 데이터의 비일치 여부를 확인한다. |
| 조직 내부 업무 통제 | 직원 근태, 일일 업무일지, 거래처, 사내정보, 전자결재, 문서보관함을 운영한다. |
| 센터/팀 간 커뮤니케이션 | 전체공지, 본부/팀별/센터별/업무별 공지, 자유게시판, 사진 게시판, 온라인미팅으로 업무 지시와 확인 댓글을 주고받는다. |
| 대외 채널 운영 지원 | 홈페이지 팝업, 접속통계, 신청서 관리, 확인서 발급 요청, 출강 거래처/강사 관리를 처리한다. |
| 자격/시험 운영 | ITT 시험관리와 확인서 발급, Certi 통합 관리를 통해 시험·자격·수료 관련 후속 업무를 지원한다. |

## 4. 관찰된 진입 및 공통 구조

### 접근 흐름

```text
https://gw.timestesol.com/
  -> JavaScript location.href='/login.php'
  -> 역할 선택(Staff / Teacher)
  -> ID / PW 입력
  -> /wl/mbr/lgnRgtr.php POST
  -> 로그인 이후 /sbMn.php?... 기반 메뉴 화면
```

로그인 화면의 폼 반환 경로 주석과 이후 HTML을 종합하면, 로그인 후에는 `sbMn.php`가 `pgNm`, `menu`, `submenu` 파라미터로 각 업무 화면을 라우팅하는 구조다. 공통 레이아웃은 상단 사용자 영역, 메뉴/회원 검색, 1차 GNB, 좌측 서브메뉴, 본문, 우측 또는 하단 연락처/내선번호 정보로 구성된다.

### 로그인 화면 구성

| 영역 | 관찰 내용 | 평가 |
| --- | --- | --- |
| 브랜드 | `TIMES MEDIA` 로고 이미지 | 회사 식별은 가능 |
| 안내 문구 | `Sign up the ID&Password` | 로그인 화면의 용도와 맞지 않고 문장도 어색함 |
| 구분 | `Staff`, `Teacher` 라디오 버튼, Staff 기본 선택 | 사용자군 구분은 명확 |
| 입력 | ID, PW 및 Login 버튼 | 최소 인증 흐름 제공 |
| 알림 | `접속IP위치 수집중입니다.` | 수집 목적·범위·고지 의미가 불명확 |

화면은 약 552px 폭의 이미지와 테이블 기반 패널이 큰 단색 배경 중앙에 놓인다. 데스크톱에서는 사용할 수 있으나 모바일·고해상도 환경에서 콘텐츠 크기와 가독성이 충분하지 않다.

### 로그인 이후 공통 요소

| 영역 | 관찰 내용 | 의미 |
| --- | --- | --- |
| 사용자/권한 영역 | 로그인 사용자명, `[관리자]`, `[쪽지팝업창]`, `[일일콜리스트]`, `[사내SMS]`, `[info관리자]`, `Log-out`, `My page`, `칭찬/벌점`, `강사관리` | 운영자 권한, 메시지, 콜 업무, SMS, 강사관리로 바로 접근하는 내부 운영 허브 |
| 통합 검색 | `메뉴`와 `회원` 검색 타입, 메뉴검색 자동완성 형태 | 기능 수가 많아 메뉴/회원 검색이 핵심 탐색 수단으로 보임 |
| 좌측 메뉴 | 선택된 1차 메뉴의 하위 기능을 패널 형태로 노출 | 각 업무 영역 안에서 다시 세부 업무를 선택 |
| 연락처/내선 | 본사, 센터, 부서 내선번호 반복 노출 | 전화 기반 협업이 강한 조직 운영 환경 |
| 도움말 링크 | 각 페이지 `페이지 도움말` 링크 | 기능별 매뉴얼 체계가 있었거나 계획된 흔적 |

## 6. 주요 기능 상세 분석

### 학생관리

학생관리 영역은 인트라넷의 핵심 업무로 보인다. `테솔`, `번역사`, `ICS`, `기타` 화면은 공통적으로 상담/예약/등록 상태를 검색하고 목록으로 관리하는 구조이며, `회계관리`와 `Certi통합관리`가 후속 회계·수료 업무를 맡는다.

| 기능 | 관찰된 항목 | 해석 |
| --- | --- | --- |
| 과정별 학생 조회 | 테솔, 번역사, ICS, 기타 | 교육 상품 또는 사업부별 학생 DB를 분리 조회 |
| 상담/예약 관리 | 첫상담, 최종상담, 최종상담인, 예약상태, 예약일, 고민상담 | 문의자 유입부터 등록 전 상담 상태를 추적 |
| 학사 상태 관리 | 졸업여부, 졸업자 포함, 강좌분류, 세부강좌, 기수/개강일 | 수강생을 강좌·기수·센터 단위로 관리 |
| 납부 관리 | 입학금, 수업료, 모든 납부금, 납부여부, 현금영수증 | 학생 상태와 회계 상태가 직접 연결됨 |
| 예외 상태 관리 | 환불자, 연기자, 전출자, Fail자, 입학금 납부 후 미등록자 | 운영 리스크가 큰 케이스를 별도 필터링 |
| 통계/검증 | 입학인원, 졸업인원, 수업료합계, 비일치, 교재비일치, 전형료비일치 | 센터 입력값과 회계/수료 데이터를 대조하는 관리 화면 |

학생관리 화면은 단순 CRM이라기보다 학원 운영용 ERP에 가깝다. 상담 유입 경로, 강좌 분류, 납부, 졸업, 확인서/자격증 발급까지 연결되어 있어 데이터 정합성이 전체 운영 품질에 직접 영향을 준다.

### 업무관리

업무관리에는 직원 개인 업무 기록과 조직 운영 기능이 모여 있다.

| 기능 | 관찰된 항목 | 해석 |
| --- | --- | --- |
| 업무일지작성 | 날짜별 업무 제목, 기간, 완료 여부, 저장 | 직원이 일 단위/기간 단위 업무를 기록 |
| 업무일지확인 | 작성된 업무일지 확인 | 관리자나 상급자가 업무 진행을 검토 |
| 근태관리(Timecards) | 출근/퇴근, 정상근무, 월2회근무, 추가근무, 대체근무, 탄력근무, 대체휴무, 공가, 연차, 반차, 결근, 비고 | 월간 캘린더형 출퇴근 및 근무 유형 관리 |
| 거래처관리 | 거래처명, 담당자, 대표자, 연락처, 팩스, 주소, 메모, 분류 | B2B 제휴/외부 거래처 등록과 검색 |
| 사내정보 | 센터, 직무, 직원 구분, 재직 상태, 키워드 검색 | 내부 인명/조직 정보 디렉터리 |
| 일일업무개별조회 | 특정 사용자/일자 기준 업무 조회 | 개인별 업무 상태 추적 |

근태 화면은 일반 출퇴근 기록뿐 아니라 추가근무, 출장, 탄력근무, 반차 등 예외 근무 규칙이 상세히 안내된다. 비정상 근무 유형에는 비고 입력을 요구하는 로직도 있어, 실제 인사/급여 또는 내부 평가 근거로 활용될 가능성이 높다.

### 전자결재

전자결재는 단순 결재함 이상의 문서 양식 기반 워크플로를 갖고 있다.

| 기능 | 관찰된 항목 | 해석 |
| --- | --- | --- |
| 결재할 문서 | 기안자, 제목, 다음결재, 작성일, 결재자 필터 | 사용자가 처리해야 할 승인 업무 |
| 수신된 문서 | 기안자, 제목 검색 | 참조/수신 문서 확인 |
| 기안하기 | 결재구분, 결재라인, 수신처, 제목, 지출결의 금액/결제수단, 임시보관함, 결재올리기 | 내부 승인 문서 작성과 상신 |
| 진행상태 | 기안진행상태, 결재진행상태, 종결/보류/반려 문서 | 작성자와 결재자 관점의 상태 추적 |
| 문서보관 | 개인문서보관함, 개인보관함, 샘플 보관함, 종류별 문서 | 완료 문서 재사용/검색 |

기안 양식에는 회의록, 업무진행 보고서, 사직서, 휴학신청서, 장학금 신청서, 성적표, 교육비 납입증명서, 재학증명서, 월별일정표, 강의결과보고서 등 교육기관 운영 문서가 포함된다. 지출결의는 금액과 결제수단 입력 검증, 증빙 보관 안내, 긴급 지출 사유 입력 같은 회계 통제 규칙을 포함한다.

### 온라인미팅/프로젝트

온라인미팅은 일반 화상회의 예약 기능이라기보다 비동기 업무 협업 게시판에 가깝다.

| 기능 | 관찰된 항목 | 해석 |

HTML만으로 권한 정책을 확정할 수는 없지만, 화면 구조상 다음 사용자군을 가정할 수 있다.

| 본부/관리자 | 메뉴 관리, 공지, 전자결재, 회계 검증, 전체 학생/센터 데이터 확인 |
| 센터 직원/매니저 | 학생 상담, 예약, 등록, 납부 확인, 센터 공지 확인, 온라인미팅 댓글 처리 |

## 8. 데이터와 업무 흐름 판단

전체 기능은 다음과 같은 운영 흐름으로 연결된다.

```text
홈페이지/광고/상담 유입
  -> 학생관리에서 문의자 등록, 상담, 예약상태 관리
  -> 강좌/센터/기수 배정
  -> 입학금/수업료 납부 확인 및 회계관리
  -> 수업 일정, 레슨플랜, 강사/출강 관리
  -> 업무일지와 온라인미팅으로 운영 이슈 처리
  -> 전자결재로 지출/문서/보고 승인
  -> 졸업, 수료, Certi, 확인서 발급
  -> 통계/공지/자료/보관함으로 기록 유지
```

즉, 인트라넷은 교육기관의 프론트오피스(상담·신청), 백오피스(회계·결재·근태), 학사 운영(강좌·수료·증명), 내부 협업(공지·미팅)을 모두 포괄한다. 리뉴얼 시 단순 화면 개선보다 업무 데이터 모델과 권한 모델을 먼저 정리해야 한다.

## 10. 보안 및 기술 관찰

| 우선순위 | 관찰 사실 | 위험 또는 검토 이유 |
| --- | --- | --- |
| High | 응답 헤더가 `Apache/2.4.57 (Unix) OpenSSL/1.0.2k-fips PHP/5.2.17`과 `X-Powered-By: PHP/5.2.17`을 노출 | 노출된 버전이 실제 운영 구성이라면 매우 오래된 PHP/OpenSSL 사용 가능성이 있으며, 공격 표면 파악도 용이 |
| High | 로그인 이후 HTML에 학생, 연락처, 회계, 미팅, 내부 계정 관련 내용이 포함됨 | 권한 통제, 다운로드/캐시/로그 보관 정책을 엄격히 점검해야 함 |
| High | 일부 JavaScript에 SQL 구문 또는 상세 쿼리성 문자열이 클라이언트에 노출되는 흔적이 있음 | 서버 내부 구조 노출 및 접근 통제 설계 점검 필요 |
| Medium | 세션 쿠키가 공개 응답에서 `PHPSESSID=...; path=/` 형태로 관찰됨 | `Secure`, `HttpOnly`, `SameSite` 속성이 응답에 보이지 않아 세션 보호 설정 점검 필요 |
| Medium | 루트 이동이 HTTP 리다이렉트가 아닌 JavaScript 이동 | 스크립트 비활성 환경과 보안/모니터링 일관성에 불리 |
| Medium | 레거시 AJAX, ActiveXObject 분기, 팝업 중심 UI가 다수 존재 | 구형 브라우저 호환 설계가 남아 있어 보안 업데이트와 현대 브라우저 정책 대응이 어려울 수 있음 |
| Medium | 파일 첨부, 팝업 이미지 업로드, 문서/디자인물 자료실 기능 존재 | 업로드 파일 확장자, MIME, 악성 파일 검사, 저장 경로 접근 통제 점검 필요 |
| Medium | 로그인 페이지가 접속 IP 위치 수집을 고지하지만 상세 설명/링크가 없음 | 개인정보 및 접속기록 처리 고지 검토 필요 |

버전 문자열은 헤더에서 관찰한 값이며 실제 패치 상태를 확정하는 자료는 아니다. 다만 운영 서비스라면 서버 정보 노출 최소화와 지원 종료 구성 여부 확인이 즉시 필요하다.

### 권고 조치

1. 서버/PHP/OpenSSL 실제 버전과 보안 패치 상태를 확인하고, PHP 5.x가 실제 사용 중이라면 지원 버전으로 이전 계획을 최우선으로 세운다.
2. 세션 쿠키 보안 속성, 로그인 시도 제한, 계정 잠금/추가 인증, 로그 보관 정책을 점검한다.
3. 학생, 회계, 연락처, 온라인미팅 상세 내용에 대한 역할별 조회 권한과 컬럼 마스킹 정책을 재정의한다.
4. 서버 버전 배너와 `X-Powered-By` 헤더 노출을 제거한다.
5. 루트 접근은 서버 측 `302` 또는 인증 상태에 따른 안전한 라우팅으로 변경한다.
6. 파일 업로드 기능은 확장자 허용목록, 바이러스 검사, 다운로드 권한, 저장소 분리, 실행 방지 설정을 점검한다.
7. 로그인 화면 개편 시 개인정보 처리 안내 및 계정 복구 경로를 포함한다.
